# 🌐 PART 1 — Web Technology II

## Chapter 1: PHP Basics

**Q: What is PHP?**
> PHP stands for Hypertext Preprocessor. It is a server-side scripting language used to create dynamic web pages. PHP code runs on the server and sends HTML to the browser.

**Q: What is $_GET?**
> $_GET collects data sent through the URL. Example: `page.php?name=Raj` → `$_GET['name']` gives "Raj". Data is visible in the URL.

**Q: What is $_POST?**
> $_POST collects data sent through an HTML form using POST method. Data is NOT visible in the URL, so it is more secure than GET.

**Q: $_GET vs $_POST difference?**
> $_GET sends data in the URL — visible, less secure, limited data size. $_POST sends data in the request body — hidden, more secure, can send large data.

**Q: What is $_SERVER?**
> $_SERVER is a PHP superglobal that stores server and environment information. Example: `$_SERVER['PHP_SELF']` gives the current file name.

**Q: What is a Session?**
> A session stores user data on the SERVER. The data is available across all pages until the browser is closed. Example: keeping a user logged in.

**Q: What is a Cookie?**
> A cookie stores small data in the CLIENT'S BROWSER. It stays even after the browser is closed until the expiry date set by the programmer.

**Q: Session vs Cookie?**
> Session is stored on the server — more secure — expires when browser closes. Cookie is stored on the browser — less secure — stays until expiry date.

**Q: How to start a session?**
> Call `session_start()` at the very top of every PHP page that uses session variables. It must be called before any HTML output.

**Q: How to set and read a Cookie?**
> Set: `setcookie('name', 'value', time()+3600)` — must be before HTML output.
> Read: `$_COOKIE['name']` gives the stored value.

**Q: What is session_destroy()?**
> It deletes all session data from the server. Used when the user logs out.

**Q: What is isset()?**
> isset() checks whether a variable is set and not NULL. Used to check if a form was submitted before processing it.

**Q: What is self-processing page?**
> A PHP page that both displays the form AND processes the submitted data. The form action is set to `$_SERVER['PHP_SELF']` so it submits to the same page.

**Q: What is PHP Error Handling?**
> Use a `try` block for code that may fail, a `catch` block to handle the error, and `throw new Exception()` to manually trigger an error.

## Chapter 2: XML

**Q: What is XML?**
> XML stands for Extensible Markup Language. It is used to store and transport data. Unlike HTML, the tags in XML are defined by the user based on the data.

**Q: XML vs HTML?**
> XML stores and describes data, tags are user-defined, and it is case-sensitive. HTML displays data on a webpage, tags are predefined, and it is not case-sensitive.

**Q: What is well-formed XML?**
> An XML document that follows all basic rules — has exactly one root element, all tags are properly opened and closed, and elements are correctly nested.

**Q: What is DOM?**
> DOM stands for Document Object Model. It represents an XML or HTML file as a tree structure. We can read, add, or change data using PHP or JavaScript through the DOM.

**Q: What is SimpleXML in PHP?**
> SimpleXML is a PHP extension that makes reading XML files very easy. The main function is `simplexml_load_file('file.xml')` which loads the XML into an object.

**Q: How to access XML elements and attributes using SimpleXML?**
> Elements: use arrow operator — `$xml->book->title`
> Attributes: use square brackets — `$xml->book['id']`

**Q: What is DOMDocument?**
> DOMDocument is a PHP class used to parse XML using the DOM approach. Useful methods are `getElementsByTagName()` and `getAttribute()`.

## Chapter 3: JavaScript and jQuery

**Q: What is JavaScript?**
> JavaScript is a client-side scripting language that runs directly in the browser. It is used to make web pages interactive and dynamic without contacting the server.

**Q: What is jQuery?**
> jQuery is a JavaScript library with the motto "Write less, do more." It simplifies DOM manipulation, event handling, and AJAX with shorter and cleaner code.

**Q: jQuery Selectors?**
> `$('p')` — selects all paragraph tags. `$('#id')` — selects element by ID. `$('.class')` — selects elements by class name.

**Q: What are DOM Events?**
> Events triggered by user actions: `onclick` (mouse click), `onload` (page load), `onmouseover` (hover), `onmouseout` (leave), `onblur` (focus lost), `onsubmit` (form submit).

**Q: alert() vs confirm() vs prompt()?**
> `alert()` shows a message — no return value. `confirm()` shows OK/Cancel — returns true or false. `prompt()` asks for user input — returns the typed text.

**Q: before() vs after() vs prepend()?**
> `before()` inserts content outside and before the element. `after()` inserts outside and after. `prepend()` inserts inside the element at the beginning.

**Q: How to validate a form in JavaScript?**
> Use the `onsubmit` event on the form. Inside the function check all input values. Write `return false` to stop the form from submitting when validation fails.

## Chapter 4: AJAX

**Q: What is AJAX?**
> AJAX stands for Asynchronous JavaScript and XML. It allows a web page to communicate with the server and update part of the page without doing a full page reload.

**Q: How does AJAX work?**
> 1. User triggers an action (like clicking a button)
> 2. JavaScript creates an XMLHttpRequest object
> 3. Request is sent to the server
> 4. Server sends back a response
> 5. JavaScript updates only the needed part of the page

**Q: What is XMLHttpRequest?**
> It is a JavaScript object used to send HTTP requests to the server and handle the response — all happening in the background without refreshing the page.

**Q: readyState values?**
> 0 = Not sent, 1 = Opened, 2 = Headers received, 3 = Loading, 4 = Done (response fully received)

**Q: What does readyState=4 and status=200 mean?**
> readyState=4 means request is complete. status=200 means the server responded successfully. We process the response only when both conditions are true.

**Q: What is onreadystatechange?**
> A function that automatically runs every time the readyState changes. Inside it we check if readyState==4 and status==200 before reading the response.

# 📊 PART 2 — Data Analytics

## Chapter 1: Introduction

**Q: What is Data Analytics?**
> Data Analytics is the science of examining raw data to discover useful patterns and insights that help in making better decisions.

**Q: Types of Analytics?**
> Descriptive — What happened? Diagnostic — Why did it happen? Predictive — What will happen? Prescriptive — What should we do about it?

**Q: What is ROC Curve and AUC?**
> ROC (Receiver-Operator Characteristic) is a graph showing classifier performance. AUC (Area Under Curve) measures the area under this graph. AUC close to 1 = very good model. AUC = 0.5 = random guessing.

---

## Chapter 2: Machine Learning

**Q: What is Machine Learning?**
> Machine Learning is teaching computers to learn patterns from data automatically, without being explicitly programmed with rules for every situation.

**Q: AI vs ML vs Deep Learning?**
> AI is the broad field of making machines intelligent. ML is a subset of AI where machines learn from data. Deep Learning is a subset of ML that uses neural networks with many layers.

**Q: Types of Machine Learning?**
> Supervised — learns from labeled data (classification, regression). Unsupervised — no labels, finds hidden patterns (clustering). Semi-supervised — mix of both. Ensemble — combines multiple models for better accuracy.

**Q: Linear vs Logistic Regression?**
> Linear Regression predicts a continuous number output like salary or house price. Logistic Regression predicts a category like yes/no or 0/1 — for example whether a person will buy a car.

**Q: What is train_test_split()?**
> A function from sklearn that splits the dataset into a training set (model learns from it) and a testing set (we check how accurate the model is).

**Q: What is fit() and predict()?**
> `fit()` trains the model using training data. `predict()` uses the trained model to generate predictions on new or test data.

**Q: What is R² score?**
> R² (R-squared) measures how well a regression model fits the data. R²=1 means perfect fit, R²=0 means the model explains nothing.

**Q: What is accuracy_score()?**
> It measures what percentage of predictions were correct. Formula: correct predictions divided by total predictions. From sklearn.metrics library.

**Q: What is Confusion Matrix?**
> A 2×2 table that evaluates a classifier. TP = predicted yes, actually yes. TN = predicted no, actually no. FP = predicted yes, actually no. FN = predicted no, actually yes.

## Chapter 3: Apriori and Association Rules

**Q: What is Market Basket Analysis?**
> Finding which products customers frequently buy together. For example — people who buy Bread also tend to buy Butter. Used by supermarkets for product placement.

**Q: Support, Confidence, Lift?**
> Support = how often the itemset appears in all transactions. Confidence = if A is bought, how likely is B also bought. Lift = if Lift > 1, A and B have a positive relationship.

**Q: What is Apriori Algorithm?**
> An algorithm to find frequent itemsets in transaction data. Its key principle is — if an itemset is not frequent, none of its supersets can be frequent either. This is used to prune candidates early and save time.

**Q: FP-Growth vs Apriori?**
> Apriori scans the database multiple times and is slow for large data. FP-Growth scans only twice, builds an FP-Tree structure, and is much faster.

**Q: What is TransactionEncoder?**
> It converts a list of transactions into a binary True/False matrix. This binary format is required as input by the Apriori function in Python's mlxtend library.

## Chapter 4: Text Analytics

**Q: What is NLP?**
> NLP stands for Natural Language Processing. It is the field of teaching computers to understand, read, and process human language — both text and speech.

**Q: What is Tokenization?**
> Splitting text into smaller units called tokens. `word_tokenize()` splits into individual words. `sent_tokenize()` splits into sentences.

**Q: What are Stop Words?**
> Extremely common words like "the", "is", "a", "an" that carry no useful meaning. They are removed before analysis so the model focuses on meaningful words only.

**Q: Stemming vs Lemmatization?**
> Stemming cuts a word to its root form — may not be a real word (e.g., "running" → "run"). Lemmatization returns the correct grammatical base form (e.g., "better" → "good"). Lemmatization is more accurate.

**Q: What is TF-IDF?**
> TF (Term Frequency) = how often a word appears in this document. IDF (Inverse Document Frequency) = how rare the word is across all documents. High TF-IDF means the word is important and unique to that document.

**Q: What is Sentiment Analysis?**
> Identifying the emotion or opinion expressed in text — whether it is Positive, Negative, or Neutral. Commonly used for analyzing product reviews and social media posts.

**Q: What is Extractive Summarization?**
> A method of summarizing text by selecting the most important sentences from the original document. No new sentences are created — it just picks the best existing ones.
